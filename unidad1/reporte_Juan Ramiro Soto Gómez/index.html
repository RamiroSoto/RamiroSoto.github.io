Resumen

La estancia 1 fue realizada dentro de las instalaciones de la SUBSECRETARIA DE INNOVACIÓN Y TECNOLOGÍAS DE LA INFORMACIÓN, ubicada dentro de la Torre Bicentenario con dirección Parque Bicentenario, Boulevard Praxedis Balboa 87083. En este reporte se mostrará lo realizado en la semana que comprende los días martes 19 de noviembre de 2019 hasta viernes 22 de noviembre de 2019. Las actividades que fueron llevadas a cabo fueron asignadas por el proyecto de “Implementación De Sistema De Administración, Control Y Repositorio De La Plataforma CAPASITI”, el cual tiene como fin mostrar a los alumnos como se desarrollan las actividades dentro de una empresa y por sobre todo poder implementar los conocimientos adquiridos durante los cuatrimestres cursados anteriormente, así como la adquisición de nuevos saberes.

Primeros pasos

El primer paso para comenzar con la práctica, fue facilitada una carpeta con la cuál acceder de manera local a la página que se deseaba modificar, esta fue ingresada como se mostró anteriormente a la carpeta www de wamp, sin embargo, se encontró que al momento de querer acceder de manera local se redirigía de manera automática a otra red.
Después de esto, haciendo uso de FileZilla, se descargaron los archivos que se encontraban en el servidor. Pero antes de esto se necesitó conectar a través de la red IP del usuario para que se le cedieran los permisos para poder editar el servidor capasiti.tamaulipas.gob.mx pasando por primera instancia conectarse a la IP de la Torre Bicentenario para poder acceder a dicho servidor.

Antes de subir la página

Como otra de las tareas encomendadas, fue necesario realizar un re direccionamiento, pero ahora en lugar de acceder al inicio se buscaba acceder a la sección de cursos que se encuentran dentro de CAPASITI. Para poder realizar esto se accedió a app->controler->components->appcontroler líneas 156-157 y se cambió la dirección.
Una de las ultimas tareas antes de poder subirlo al servidor fue cambiar las opciones de Login y Logout, es decir, las opciones que redirigen a una u otra parte de la página al ingresar un usuario, esto se encuentra en app->components->appController en $this->Auth-> loginAction = '/usuarios/login/'; logoutRedirect = '/usuarios/login/'; loginRedirect = '/pages/index/'.

Subiendo la página

Con la plataforma en funcionamiento, se comenzó con la depuración de la base de datos, esto gracias a los permisos de administrador que se consiguieron, siendo más fácil la eliminación de elementos no necesarios, como lo eran algunos usuarios y cursos que se encontraban, dejando solo un grupo de usuarios que funcionan como pruebas para revisar si cada uno tiene sus permisos correspondientes.
Se encontró que el error surgía debido a que en el código ubicado en UsuariosCalendarizadosController.php en las líneas 591 a la 595 se hacía mal uso de una palabra reservada, además de que había palabras mal escritas que hacían referencia a tablas de la base de datos, por lo cual nunca se podía acceder a la misma y generar los reportes.

Para terminar

Ahora, con las nuevas actualizaciones, subidas utilizando nuevamente FileZilla, se realizaron algunas pruebas con personas ajenas al proyecto, esto con el fin de observar que tan didáctica es la página y lograr conseguir cualquier otro error que pueda surgir y que el equipo no haya encontrado con anterioridad.
Los errores encontrados fueron solucionados y se termino con los cambios realizados, haciendo entrega del trabajo en su totalidad

Herramientas

Wampserver.

Se trata de un servidor cuyas principales características son su funcionalidad con la base de datos de MySQL, el uso de apache, y su uso con el lenguaje PHP, debido a los puntos anteriores y a su fácil implementación fue elegido para poder desarrollar la practica requerida.

MySQL.

MySQL es un software creado por la empresa sueca MySQL AB por David Axmark, Allan Larsson y Michael Widenius. Su lenguaje de programación se trata de Structured Query Lenguaje(SQL) desarrollado en 1981 por IBM. Como muchos otros sistemas de gestión MySQL cuenta con elementos que apoyan al mantenimiento y uso de las bases de datos, tales como datos relacionales, integridad referencial y transacciones.

FileZilla.

Es una aplicación desarrollada en 2001 por Tim Koose, tiene la funcionalidad de descargar páginas web de su respectivo servidor, siempre y cuando se cuente con la información de esta misma. Para el desarrollo de las estancias fue indispensable debido a que se requería descargar todos los paquetes de un servidor a partir del cual se realizarían cambios y se vería como es su comportamiento.

Atom

Se trata de un editor de texto, el cual funciona para varios tipos de equipos. Fue creado por GitHub el 26 de febrero de 2014.
